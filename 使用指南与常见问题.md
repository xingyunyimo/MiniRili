# MiniRili 使用指南与常见问题

## 一、快速上手

### 安装 App
1. 从 GitHub Releases 页面下载最新版 APK
2. 在手机上打开 APK 文件安装
3. 首次打开会显示隐私政策，阅读后点击"同意"进入应用

### 界面导航

应用底部导航栏提供四个主要入口：
- **日历** — 主界面，默认月视图
- **天气** — 天气详情页
- **所有事件** — 你的全部事件列表
- **新建事件** — 快速创建新事件

## 二、功能详解

### 📅 日历视图

顶部工具栏可切换四种视图模式：

| 视图 | 说明 | 推荐用途 |
|---|---|---|
| 月视图 | 默认视图，每格显示公历+农历日期、事件圆点 | 概览整月 |
| 周视图 | 横向一周，下方显示当日事件列表 | 看本周安排 |
| 日视图 | 以小时为维度的双列时间轴布局，左侧为事件列（占用具体时间段），右侧为条目列（如全天事件/节假日）。当日事件在界面上下方分为"时间事件"列表和"全天事件"列表显示。 | 查看某天详细安排 |
| 年视图 | 12 个月缩略排列，点击某月跳转到月视图 | 快速定位月份 |

**操作：**
- **翻页** — 点击左右箭头或左右滑动切换上/下月（周/日）
- **跳转** — 点击顶部日期范围可弹出日期选择器，快速跳转到任意日期
- **回到今天** — 点击顶部"今天"按钮
- **双击日期** — 在月视图中双击某天，切换到该天的日视图

### 📝 事件与提醒

**新建事件：**
1. 点击底部导航栏"+"按钮，或月视图中点击某天的空白区域
2. 填写标题（必填）、描述、选择日期和时间
3. 设置提醒：选择提醒时间（事件开始时/提前5/10/15/30分钟/1小时/1天/2天/1周）
4. 可选：设置标签和颜色，便于分类

**重复事件：**
- 在事件编辑页面选择重复类型：每日/每周/每月/每年/工作日/周末
- 农历模式可选：每月（农历）/每年（农历）
- 重复事件可单独跳过某次触发：在事件详情页点击"跳过明天"按钮

**查看/编辑事件：**
- 在日历上点击事件圆点 → 进入事件详情页
- 详情页可编辑事件内容、查看提醒状态
- 支持长按事件列表中的条目进行拖拽排序

**事件支持的功能：**
- 优先级标记
- 完成状态切换
- 标签分类与颜色标记
- 附件添加

### 🌤️ 天气

天气详情页包含多个卡片模块：
- **当前天气** — 温度、天气图标、天气描述
- **空气质量（AQI）** — 空气质量指数及等级
- **生活指数** — 紫外线、湿度、气压、能见度、云量、风速风向
- **出行建议** — 基于天气规则引擎生成的场景化提醒（高温/降雨/大风/温差等）
- **24 小时预报** — 逐小时气温+天气图标
- **15 天趋势** — 每日最高/最低气温折线图+天气图标
- **日出日落** — 当日及次日日出日落时间+月相

**城市管理：**
- 天气页顶部显示当前城市
- 点击"添加城市"按钮搜索并添加关注的城市
- 左右滑动切换城市
- 点击"删除"按钮移除城市
- 点击定位按钮使用 GPS 获取当前位置天气

**定位权限：**
- 首次进入天气页时，若未授权定位且未添加城市，会显示引导卡片
- 点击"允许定位"按钮，授予位置权限
- 也可以手动搜索城市添加，无需定位权限

### 🔔 每日天气通知

应用支持每日定时推送天气通知：
- 默认推送时间：每日 08:00
- 推送内容：今日天气概况 + 出行建议
- 在天气页的"出行建议"卡片右上角齿轮按钮中可配置：
  - 开关：启用/关闭每日通知
  - 时间：选择推送的小时和分钟

### 🏷️ 节假日

- 中国法定节假日以红色标记在日历上显示
- 调休补班日以特殊标记显示
- 数据基于国务院每年发布的安排，内置 2021–2026 年数据
- 节假日信息无需联网，离线可用

### 💾 数据导入导出

**导出：**
1. 在日历页点击菜单 → 导出 ICS 或导出 JSON
2. 选择保存位置
3. 所有事件将导出为文件

**导入：**
1. 在日历页点击菜单 → 导入 ICS 或导入 JSON
2. 选择之前导出的文件
3. 事件将自动导入应用

**格式说明：**
- ICS：标准日历交换格式（RFC 5545），可导入 Google Calendar、Outlook 等其他日历应用
- JSON：MiniRili 专属格式，包含完整的应用内事件信息

### 📱 桌面小部件

应用提供了一种桌面小部件：
- **4×3 组合小部件**：同时显示农历信息、今日天气、近期事件列表

**添加方法：**
Android 桌面长按空白处 → 小部件 → 找到 MiniRili → 添加组合小部件。

## 三、权限说明

| 权限 | 用途 | 是否必须 |
|---|---|---|
| 通知（POST_NOTIFICATIONS） | 发送事件提醒和每日天气通知 | 推荐开启 |
| 位置（ACCESS_FINE_LOCATION / ACCESS_COARSE_LOCATION） | 获取当前位置天气 | 可选 |
| 精确闹钟（SCHEDULE_EXACT_ALARM） | 确保提醒准时触发 | 推荐开启 |

## 四、常见问题

### Q1：收不到事件提醒？

**可能原因及解决方法：**

1. **通知权限未开启**（最常见）
   - 前往手机"设置" → "应用管理" → "MiniRili" → "通知管理"
   - 确保通知开关已打开
   - 首次安装时应用会弹出权限请求，请选择"允许"

2. **电池优化限制了应用后台**
   - 前往手机"设置" → "应用管理" → "MiniRili" → "电池"
   - 选择"无限制"或"不优化"
   - 部分厂商（华为/小米/Oppo/Vivo）需额外在"自启动管理"中允许应用后台运行
   - 应用内提供了厂商引导：日历页菜单 → "电池优化" → 查看各厂商设置说明

3. **精确闹钟权限未开启**（Android 13+）
   - 前往"设置" → "应用管理" → "MiniRili" → "特殊权限" → "闹钟和提醒"
   - 确保已允许

4. **手机重启后未打开应用**
   - 重启手机后，应用会自动恢复已注册的提醒
   - 如果提醒仍未触发，可以手动打开一次应用

### Q2：天气数据不显示或显示异常？

1. **检查网络连接** — 天气数据需要联网获取
2. **定位未开启** — 如果使用当前位置，确保定位权限已授予
3. **手动添加城市** — 也可以点击"添加城市"搜索城市手动添加
4. **数据缓存** — 天气数据缓存 30 分钟，不会每次打开都刷新
5. **15天限制** — Open-Meteo 免费 API 最多提供 15 天预报，超出范围会回退到今日数据

### Q3：日期显示不对或农历显示异常？

- 应用基于 Android 内置的 ICU4J ChineseCalendar，覆盖 1900–2200 年
- 如果系统语言设置为中文，农历会自动显示
- 如果系统日期设置异常，请检查手机"设置" → "日期和时间"是否为自动

### Q4：导入 ICS 文件失败？

1. 确保文件编码为 UTF-8
2. 确保文件符合 RFC 5545 标准格式
3. 导入大文件时请耐心等待
4. 如果仍然失败，可以尝试导出 JSON 再导入

### Q5：可以备份事件数据吗？

可以。使用导出功能（ICS 或 JSON）将数据保存到手机存储，之后可以通过导入恢复。
建议定期备份，特别是在系统重置或换手机之前。

### Q6：应用需要哪些权限？安全吗？

应用**不需要任何账户**，**不需要联网注册**，**不上传任何个人数据**。
所有数据（事件、设置）仅存储在本地设备上。
天气数据通过直接调用 Open-Meteo 公共 API 获取，不经过第三方服务器中转。

### Q7：如何删除所有事件？

目前没有一键清空功能，需要在"所有事件"列表中逐条删除，或通过导入空文件覆盖。

### Q8：桌面小部件不更新？

- 确保应用没有被系统杀死
- 部分厂商对小部件刷新频率有限制
- 可以尝试移除小部件后重新添加
- 如果事件变更了，切换到其他视图再切回来触发刷新

### Q9：为什么是 debug 版本？

当前 Release 提供的是 debug 构建的 APK，安装时系统会提示"未知来源"或"此应用为调试版本"。这不影响正常使用。

后续版本会提供 release 签名版本消除此提示。

### Q10：应用支持平板吗？

应用未针对平板做专门适配，但在平板上可以正常运行。建议在竖屏模式下使用。

## 五、隐私说明

- MiniRili **不收集任何用户数据**
- 所有事件数据仅存储在本地设备 Room 数据库中
- 天气数据请求直接发送到 Open-Meteo API，不经任何中间服务器
- 应用无需创建账户、无需登录
- 详细隐私政策见应用内"隐私政策"页面
